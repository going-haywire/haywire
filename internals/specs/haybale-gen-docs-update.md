# haybale-gen-docs skill update

The `haybale-gen-docs` skill currently:

- Reads hand-authored content from `LIBRARY.md` (old name).
- Generates `OVERVIEW.md` only.

This spec updates it to:

- Read hand-authored content from `NOTES.md` (new name).
- Generate **both** `README.md` (package root) and `OVERVIEW.md` (module dir) in a single pass.
- Preserve the `<!-- marketstall:share-url:start --> ... <!-- marketstall:share-url:end -->` marker block when regenerating `README.md`, so `haywire share --save` and the generator coexist.

The target state for the generated documentation files is described in
[`docs/components/haybale-package/haybale-package-canon.md`](../../docs/components/haybale-package/haybale-package-canon.md)
§3 "Documentation files generated by `haywire-gen-docs`". The marker coordination
contract is in [`marketstall-distribution.md`](archive/marketstall-distribution.md) §6.6.

## Changes to the skill

### 1. Filename rename

Every reference to `LIBRARY.md` in the skill becomes `NOTES.md`. The skill:

- Reads `NOTES.md` (not `LIBRARY.md`) for the hand-authored "what & why" content.
- Never modifies or overwrites `NOTES.md`.
- Prepends `NOTES.md` content verbatim to the generated `README.md`.
- Appends `NOTES.md` content as an "Additional Notes" section in `OVERVIEW.md`.

If `NOTES.md` is absent, the generator emits the catalog-only versions of both files without the prepended/appended block.

### 2. README.md generation

`README.md` is fully generated. Authors who hand-edit it lose their edits on the next run. Hand-authored content belongs in `NOTES.md`.

Generated structure (per the canon doc §3):

```text
README.md
│
├── [NOTES.md content verbatim, if present]
│
├── <!-- marketstall:share-url:start -->
│   `<share URL>`                              ← managed by `haywire share --save`; never edited by generator
│   <!-- marketstall:share-url:end -->
│
└── [Component catalog — identical content to OVERVIEW.md]
```

The marker block is **inserted by the generator on first run** (when no existing README is present) with a placeholder:

```markdown
<!-- marketstall:share-url:start -->
*Subscribe URL not yet published — run `haywire share --save` after pushing this repo to a git remote.*
<!-- marketstall:share-url:end -->
```

On subsequent runs, the generator **preserves whatever is between the markers** (the placeholder, or a real URL written by `haywire share --save`). Only the content outside the markers is regenerated.

### 3. OVERVIEW.md generation

Unchanged in shape. Catalog-only content; ships inside the wheel.

If `NOTES.md` exists, append its content as a final "Additional Notes" section.

### 4. Per-component docs

Unchanged. The skill continues to generate `docs/nodes/<ClassName>.md` and `docs/widgets/<ClassName>.md` with sha256 hash tracking to skip unchanged docs.

### 5. Migration handling for existing libraries

Libraries that currently have a `LIBRARY.md` should have it renamed to `NOTES.md` on next generator run. The skill detects the old filename and either:

- Prompts the user to confirm the rename, then `git mv`s the file, OR
- Renames silently with a log message.

Single barn library currently has a `LIBRARY.md`: `barn/haybale-visiongraph/haybale_visiongraph/LIBRARY.md`. After the rename ships, that file becomes `NOTES.md`.

## Acceptance criteria

- Running the skill on a library with `LIBRARY.md` renames it to `NOTES.md` (or instructs the user to).
- Running the skill on a library with `NOTES.md` produces both `README.md` (with prepended NOTES content and marker block) and `OVERVIEW.md` (with appended NOTES content).
- Running the skill on a library with no `NOTES.md` produces catalog-only `README.md` (with marker block) and `OVERVIEW.md`.
- Running the skill on a library where `README.md` already has a marker block preserves the block's contents verbatim across runs.
- `haywire share --save` writing a URL into the marker block survives the next generator run.

## Non-goals

- No template customisation. The README/OVERVIEW shape is fixed by this spec.
- No support for hand-authored README content outside the marker block. Hand-authored content goes in `NOTES.md`.
- No coordination with non-haywire publishing tools that might also expect to edit `README.md` between markers. The marker namespace is `marketstall:`; other tools should pick distinct prefixes.
